# Vergil's Nightmare

Учебный проект на **Python + pygame**: 2D arena slasher на одном экране, вдохновлённый боевой системой Devil May Cry. Игрок управляет Вергилием, сражается с волнами демонов, переключается между **Yamato** и **Beowulf**, поддерживает комбо и повышает рейтинг стиля от **E** до **S**.

## Идея проекта

**Vergil's Nightmare** — это 2D слэшер на одной арене. Игрок должен уничтожить несколько волн врагов, используя два оружия, телепортацию, прыжки, воздушные атаки и заряжаемую способность.

Главная цель геймплея — не просто победить всех врагов, а играть эффективно: поддерживать серию ударов, не получать урон, выбирать подходящее оружие и повышать рейтинг стиля.

## Основные механики

- Арена на одном экране без больших уровней и прокрутки карты.
- Два оружия:
  - **Yamato** — быстрые атаки, удобное продолжение комбо, воздушный бой.
  - **Beowulf** — медленные, но сильные удары, эффективные против живучих врагов.
- Три типа врагов:
  - **Slow Enemy** — медленный базовый противник.
  - **Fast Enemy** — быстрый, но менее живучий противник.
  - **Heavy Enemy** — живучий враг с высоким сопротивлением подбрасыванию.
- Наземные и воздушные атаки.
- Телепортация к ближайшему врагу.
- Заряжаемая способность для сильной атаки по арене.
- Провокация: длительная рискованная анимация, успешное завершение которой увеличивает стиль.
- Система комбо и рейтинга стиля от **E** до **S**.
- Волны врагов с постепенным ростом сложности.
- Сохранение прогресса в JSON-файл.

## Управление

При первом запуске создаётся файл `saves/controls.json`. Все изменяемые кнопки управления сохраняются в нём и загружаются при следующем запуске. Значения по умолчанию описаны в `src/core/control_constants.py`, а во время игры используются обычные обращения к словарю, поэтому получение назначенной клавиши выполняется за `O(1)`.

| Действие | Клавиша по умолчанию | Можно изменить в меню |
| --- | --- | --- |
| Движение вверх | `W` | да |
| Движение влево | `A` | да |
| Движение вниз | `S` | да |
| Движение вправо | `D` | да |
| Прыжок | `Alt` | да |
| Атака | `J` | да |
| Смена оружия | `Q` | да |
| Способность | `H` | да |
| Телепортация | `K` | да |
| Провокация | `I` | да |
| Пауза | `Escape` | нет, только отображается |

Кнопка **Настройки** в главном меню открывает поверх меню панель управления в два столбца. Первый столбец называется **Передвижение**: в нём находятся движение и прыжок. Во втором столбце находятся атака, смена оружия, способность, телепортация, провокация и видимая, но неизменяемая пауза `Escape`.

## Критерии готовности

Проект считается готовым к учебной демонстрации, если выполнены следующие условия:

- Есть главное меню с пунктами **Начать игру**, **Настройки**, **Выход**.
- Игрок может пройти полный цикл: меню → бой → победа или поражение → возврат в меню.
- Реализованы 2 оружия: Yamato и Beowulf.
- Реализованы минимум 3 типа врагов.
- Реализованы минимум 5 волн врагов.
- Работают здоровье, урон, атаки, подбрасывание и смерть врагов.
- Работает система комбо и рейтинга стиля.
- Работает провокация: длительная анимация награждает стилем только после полного завершения.
- Работает сохранение и загрузка прогресса и пользовательских настроек управления из JSON.
- HUD показывает здоровье, оружие, заряд способности, комбо, стиль и номер волны.
- Есть unit-тесты для ключевой игровой логики.
- Игра поддерживает не менее 30 FPS при типовой нагрузке.

## Архитектура

Планируется модульная объектно-ориентированная архитектура с разделением на сцены, сущности и системы.

### Ключевые компоненты

| Компонент | Ответственность |
| --- | --- |
| `Game` | Инициализация pygame, главный цикл, переключение сцен. |
| `Scene` | Базовый интерфейс для меню, боя, паузы, победы и поражения. |
| `Entity` | Базовая игровая сущность с позицией, скоростью и прямоугольником столкновения. |
| `Player` | Состояние игрока, движение, атаки, оружие, способность, телепортация. |
| `Enemy` | Базовое поведение врагов, здоровье, атака, преследование игрока. |
| `Weapon` | Параметры оружия и набор доступных атак. |
| `Attack` | Урон, длительность, hitbox, активные кадры, подбрасывание и отталкивание. |
| `ComboSystem` | Подсчёт комбо, таймер сброса, рейтинг стиля. |
| `WaveManager` | Генерация волн по бюджету сложности. |
| `SaveSystem` | Сохранение и загрузка прогресса в JSON. |
| `ControlSettings` | Загрузка, валидация и сохранение назначенных клавиш управления. |
| `HUD` | Отображение игровой информации. |

### Почему выбрана такая архитектура

Такая архитектура удобна для проекта, потому что она отделяет игровой цикл от конкретных игровых механик. Сцены отвечают за состояние приложения, сущности хранят данные игровых объектов, а системы управляют отдельными правилами: волнами, столкновениями, комбо и сохранениями.

## Алгоритмы

### 1. Система комбо и рейтинга стиля

`ComboSystem` отслеживает успешные попадания, время с последней атаки, получение урона и текущий рейтинг стиля.

Планируемые ранги:

| Комбо | Ранг |
| ---: | --- |
| 0–4 | нет ранга |
| 5–9 | E |
| 10–14 | D |
| 15–24 | C |
| 25–39 | B |
| 40–59 | A |
| 60+ | S |

Алгоритм нужен, чтобы мотивировать игрока поддерживать темп боя, использовать разные атаки и избегать урона.

### 2. Генератор волн по бюджету сложности

`WaveManager` создаёт волны не только вручную, но и на основе бюджета сложности. Каждый тип врага имеет стоимость, а бюджет волны растёт по мере прохождения.

Пример:

| Тип врага | Стоимость |
| --- | ---: |
| Slow Enemy | 1 |
| Fast Enemy | 2 |
| Heavy Enemy | 4 |

Такой подход позволяет постепенно повышать сложность и при этом сохранять баланс.

## Установка и запуск

> На текущем этапе создана только стартовая структура проекта. Реализация игрового цикла будет добавлена позже.

1. Создать виртуальное окружение:

```bash
python -m venv .venv
```

2. Активировать окружение:

```bash
# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Запустить главное меню:

```bash
python main.py
```

## Тестирование

Планируемый запуск тестов:

```bash
pytest
```

На раннем этапе тесты будут покрывать чистую игровую логику без запуска окна pygame:

- расчёт ранга стиля;
- сброс комбо;
- генерацию волн;
- сохранение и загрузку JSON;
- загрузку, изменение и сохранение настроек управления;
- выбор оружия и атаки.

Фокус проекта — законченный учебный прототип с понятной архитектурой, боевой системой, волнами, комбо, сохранениями и тестами.
